## Core API

### Entity

```python
from autoflow.context_graph.core import Entity
```

A node in the context graph representing a real-world object.

**Constructor:**

```python
Entity(
    type: str,
    properties: dict[str, Any],
    id: str = "<auto-generated>",
    embedding: Optional[list[float]] = None,
    created_at: datetime = "<now>",
    _validate: Optional[bool] = None
) -> Entity
```

**Parameters:**

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `type` | str | Yes | - | Entity type (e.g., "brand", "campaign"). Must be in allowlist unless validation disabled. |
| `properties` | dict[str, Any] | Yes | - | Key-value metadata. Keys must match `^[a-zA-Z_][a-zA-Z0-9_]*$` unless validation disabled. |
| `id` | str | No | Auto-generated | Unique identifier. Format: `"entity:{uuid[:12]}"` |
| `embedding` | list[float] | No | None | Vector embedding for semantic search |
| `created_at` | datetime | No | Now | Creation timestamp (Pydantic `Field(default_factory=datetime.utcnow)`) |
| `_validate` | bool | No | None | If None, uses global config. If False, skips validation for this entity. |

**Returns:** `Entity` instance (Pydantic `BaseModel`)

**Raises:**
- `ValueError`: If `type` is empty or not in allowlist (Pydantic validation error)
- `ValueError`: If property keys are invalid (Pydantic validation error)
- `ValueError`: If too many properties (default max: 100)

**Pydantic Features:**
- Automatic validation on assignment (`validate_assignment=True`)
- JSON serialization via `model_dump_json()` or `to_dict()`
- Field validation using `field_validator` decorators
- Post-construction validation using `model_validator`
- Extra fields are forbidden (`extra="forbid"`)

**Example:**

```python
# Basic entity
entity = Entity(
    type="brand",
    properties={"name": "Nike", "vertical": "Apparel"}
)
print(entity.id)  # "entity:a1b2c3d4e5f6"

# With custom ID
entity = Entity(
    type="brand",
    properties={"name": "Nike"},
    id="brand:nike"
)

# Disable validation for this entity
entity = Entity(
    type="custom_type",  # Not in default allowlist
    properties={"name": "Custom"},
    _validate=False
)
```

**Methods:**

```python
entity.get(key: str, default: Any = None) -> Any
```
Get a property value.

```python
entity.set(key: str, value: Any) -> None
```
Set a property value.

```python
entity.label -> str
```
Human-readable label (returns `name` or `title` property if available).

```python
entity.to_dict() -> dict[str, Any]
```
Convert to dictionary representation.

```python
@classmethod
Entity.from_dict(data: dict[str, Any]) -> Entity
```
Create entity from dictionary.

### Relationship

```python
from autoflow.context_graph.core import Relationship
```

A directed edge between two entities.

**Constructor:**

```python
Relationship(
    from_entity: str,
    to_entity: str,
    type: str,
    properties: dict[str, Any] = {},
    confidence: Optional[float] = None,
    created_at: datetime = "<now>",
    _validate: Optional[bool] = None
) -> Relationship
```

**Parameters:**

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `from_entity` | str | Yes | - | Source entity ID |
| `to_entity` | str | Yes | - | Target entity ID |
| `type` | str | Yes | - | Relationship type (e.g., "competes_with"). Must be in allowlist. |
| `properties` | dict[str, Any] | No | `{}` | Metadata about the relationship |
| `confidence` | float | No | None | Confidence score (0.0 to 1.0) for probabilistic edges |
| `created_at` | datetime | No | Now | Creation timestamp |
| `_validate` | bool | No | None | If None, uses global config. If False, skips validation. |

**Returns:** `Relationship` instance

**Raises:**
- `ValueError`: If `from_entity` or `to_entity` is empty
- `ValueError`: If `type` is empty or not in allowlist
- `ValueError`: If `confidence` is not between 0 and 1

**Example:**

```python
# Basic relationship
rel = Relationship(
    from_entity="brand:nike",
    to_entity="brand:adidas",
    type="competes_with",
    properties={"intensity": "high"},
    confidence=0.95
)

# Without validation
rel = Relationship(
    from_entity="a",
    to_entity="b",
    type="custom_rel_type",
    _validate=False
)
```

**Methods:**

```python
relationship.label -> str
```
Human-readable label (returns `label` property or formatted type).

```python
relationship.to_dict() -> dict[str, Any]
```
Convert to dictionary representation.

### ContextGraph

```python
from autoflow.context_graph.core import ContextGraph
```

Main interface for the Context Graph Framework.

**Constructor:**

```python
ContextGraph(
    backend: GraphBackend,
    domains: Optional[list[ContextDomain]] = None

```

**Parameters:**

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `backend` | GraphBackend | Yes | - | Storage backend (Neo4j, InMemory, etc.) |
| `domains` | list[ContextDomain] | No | `[]` | Domain modules for validation/extraction |

**Example:**

```python
from autoflow.context_graph.backends import InMemoryBackend
from autoflow.context_graph.domains.brand import BrandDomain

graph = ContextGraph(
    backend=InMemoryBackend(),
    domains=[BrandDomain()]

```

**Methods:**

**Parameters:**
- `entity` (Entity): Entity to add

**Returns:** Entity ID

**Raises:**
- `ValueError`: If entity validation fails

**Example:**
```python

entity_id = graph.add_entity(entity)
```

#### add_relationship

```python
add_relationship(relationship: Relationship) -> str
```

Add a relationship to the graph.

**Parameters:**
- `relationship` (Relationship): Relationship to add

**Returns:** Relationship ID

**Example:**
```python

    from_entity="brand:nike",
    to_entity="brand:adidas",
    type="competes_with"
)
rel_id = graph.add_relationship(rel)
```

#### get_entity

```python
get_entity(entity_id: str) -> Optional[Entity]
```

Get an entity by ID.

**Parameters:**
- `entity_id` (str): Entity ID

**Returns:** Entity or None

**Example:**
```python
entity = graph.get_entity("brand:nike")
if entity:
    print(entity.label)
```

#### get_neighbors

```python
get_neighbors(
    entity_id: str,
    relationship_type: Optional[str] = None,
    direction: str = "outgoing"
) -> list[tuple[Entity, Optional[Relationship]]]
```

Get neighboring entities.

**Parameters:**
- `entity_id` (str): Starting entity ID
- `relationship_type` (str, optional): Filter by relationship type
- `direction` (str): "outgoing", "incoming", or "both"

**Returns:** List of (entity, relationship) tuples

**Example:**
```python
# Get all competitors
neighbors = graph.get_neighbors(
    "brand:nike",
    relationship_type="competes_with",
    direction="outgoing"
)

for entity, rel in neighbors:
    print(f"{entity.label} (intensity: {rel.properties['intensity']})")
```

#### traverse

```python
traverse(
    start_entity_id: str,
    pattern: TraversalPattern,
    max_hops: Optional[int] = None
) -> Subgraph
```

Execute a multi-hop traversal.

**Parameters:**
- `start_entity_id` (str): Starting entity ID
- `pattern` (TraversalPattern): Traversal pattern
- `max_hops` (int, optional): Override max hops from pattern

**Returns:** Subgraph containing results

**Example:**
```python
# Find brands Nike competes with, and publishers they use
pattern = TraversalPattern(
    pattern="-[:COMPETES_WITH]->(brand)-[:ADVERTISED_ON]->(publisher)",
    max_hops=2
)

subgraph = graph.traverse("brand:nike", pattern)

print(f"Found {len(subgraph.entities)} entities")
for entity_id, entity in subgraph.entities.items():
    print(f"  - {entity.label} ({entity.type})")
```

#### search_by_property

```python
search_by_property(
    entity_type: str,
    property_name: str,
    property_value: Any
) -> list[Entity]
```

Search for entities by property value.

**Parameters:**
- `entity_type` (str): Entity type to search
- `property_name` (str): Property name
- `property_value` (Any): Value to match

**Returns:** List of matching entities

**Example:**
```python
# Find all premium brands
brands = graph.search_by_property(
    entity_type="brand",
    property_name="tier",
    property_value="premium"
)
```

#### get_context_for_llm

```python
get_context_for_llm(
    start_entity_id: Optional[str] = None,
    subgraph: Optional[Subgraph] = None,
    max_hops: int = 3,
    include_paths: bool = True
) -> str
```

Get LLM-ready context from the graph.

**Parameters:**
- `start_entity_id` (str, optional): Traverse from this entity
- `subgraph` (Subgraph, optional): Use this subgraph directly
- `max_hops` (int): Max hops if traversing
- `include_paths` (bool): Include traversal paths

**Returns:** Natural language description

**Example:**
```python
# Get context for LLM
context = graph.get_context_for_llm(
    start_entity_id="brand:nike",
    max_hops=2
)

# Use in LLM prompt
prompt = f"""
Based on the following context, answer the question:

{context}

Question: Why might Nike's campaign be underperforming compared to competitors?
"""
```

#### close


### TraversalPattern

```python
from autoflow.context_graph.core import TraversalPattern
```

A pattern for multi-hop graph traversals.

**Constructor:**

```python
TraversalPattern(
    pattern: str,
    max_hops: int = 4,
    filter_properties: dict[str, Any] = {}
) -> TraversalPattern
```

**Parameters:**

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `pattern` | str | Yes | - | Cypher-like pattern (e.g., `"-[:REL_TYPE]->(type)"`) |
| `max_hops` | int | No | 4 | Maximum hops (enforced by validator) |
| `filter_properties` | dict | No | `{}` | Property filters (not yet implemented) |

**Example:**

```python
# Simple: any relationship, up to 3 hops
pattern = TraversalPattern(pattern="-[*]->", max_hops=3)

# Specific: brands that compete, and what they advertise on
pattern = TraversalPattern(
    pattern="-[:COMPETES_WITH]->(brand)-[:ADVERTISED_ON]->(publisher)",
    max_hops=2
)
```

## Authorization

### RBACAuthorizer

Role-Based Access Control.

```python
RBACAuthorizer(
```

**Default Roles:**

| Role | Permissions |
|------|-------------|
| `admin` | All permissions |
| `editor` | All except MANAGE_SCHEMA, MANAGE_USERS |

**Example:**

```python

        Permission.TRAVERSE,
        Permission.EXPORT_DATA
    }
}
authorizer = RBACAuthorizer(role_permissions=custom_roles)

# Check authorization
if authorizer.can(context, Permission.READ_ENTITY):
    # Allow
    pass
```

### ABACAuthorizer

```python
from autoflow.context_graph.auth import ABACAuthorizer
```

Attribute-Based Access Control with policies.

**Constructor:**

```python
ABACAuthorizer() -> ABACAuthorizer
```

**Methods:**

```python
add_policy(policy: Callable[[AuthContext, Permission, Any], bool]) -> None
```

Add a policy function.

**Example:**

```python
authorizer = ABACAuthorizer()

# Policy: Premium tier can delete
authorizer.add_policy(
    lambda ctx, perm, res: (
        perm == Permission.DELETE_ENTITY and
        ctx.attributes.get("tier") == "premium"
    )
)

# Policy: Users can only read their department's entities
authorizer.add_policy(
    lambda ctx, perm, res: (
        perm == Permission.READ_ENTITY and
        isinstance(res, dict) and
        res.get("department") == ctx.attributes.get("department")
    )
)

# Check
if authorizer.can(context, Permission.DELETE_ENTITY):
    # Allowed by policy
    pass
```

```python
from autoflow.context_graph.audit import AuditContextManager
```

Context manager for automatic audit logging with timing.

**Constructor:**

```python
AuditContextManager(
    auditor: Auditor,
    event_type: AuditEventType,
    operation: Optional[str] = None,
    user_id: Optional[str] = None,
    resource_type: Optional[str] = None,
    resource_id: Optional[str] = None
) -> AuditContextManager
```

**Example:**

```python
auditor = Auditor(enabled=True)

with AuditContextManager(
    auditor=auditor,
    event_type=AuditEventType.GRAPH_TRAVERSE,
    operation="traverse",
    user_id="user_001",
    resource_id="brand:nike"
):
    # Do work
    result = graph.traverse("brand:nike", pattern)
```

## Best Practices

### 1. Always Use Context Managers for Backends

```python
# Good: Auto-closes
with Neo4jBackend(uri="...") as backend:
    graph = ContextGraph(backend=backend)
    # ... use graph ...

# Bad: Manual close
backend = Neo4jBackend(uri="...")
graph = ContextGraph(backend=backend)
# ... use graph ...
backend.close()  # Easy to forget
```

### 2. Validate Early

```python
# Validate before adding to graph
entity = Entity(type="brand", properties={...})  # Validates in __post_init__
entity_id = graph.add_entity(entity)  # Already validated
```

### 3. Use Domain Modules

```python
# Good: Uses domain validation
graph = ContextGraph(
    backend=backend,
    domains=[BrandDomain(), CampaignDomain()]
)

# Bad: No domain validation
graph = ContextGraph(backend=backend)
```

### 4. Log Everything in Production

```python
auditor = Auditor(enabled=True)  # Enable in production

# Log all operations
auditor.log_read("entity", entity_id, user_id)
auditor.log_write("entity", entity_id, user_id)
```

### 5. Use Async Audit in Production

```python
from autoflow.context_graph.audit import AsyncAuditBackend

auditor = Auditor(
    backend=AsyncAuditBackend(
        backend=DatabaseAuditBackend(db_url),
        queue_size=50000
    )
)
```

## Error Handling

### Common Errors

| Error | Cause | Solution |
|-------|-------|----------|
| `ValueError: Invalid entity type 'X'` | Type not in allowlist | Use `config.allow_entity_type('X')` or `_validate=False` |
| `ValueError: Invalid property key 'X'` | Key doesn't match `^[a-zA-Z_][a-zA-Z0-9_]*$` | Use valid key names or disable validation |
| `ValueError: Confidence must be between 0 and 1` | Confidence outside range | Use value between 0.0 and 1.0 |
| `ImportError: Neo4j backend requires 'neo4j'` | Package not installed | `pip install autoflow[neo4j]` |
| `ImportError: JWTAuthenticator requires 'pyjwt'` | Package not installed | `pip install pyjwt` |
